Case Study
Data Engineer m/f/d
Please prepare a case study for the interview on the following topic:
Please imagine that you are a ‘Data Engineer (m/f/d)’ in the organisational unit Smart Home / Electronics (SH/EC), department ‘AI & Data’. In this role, you build the pipelines for data products based on IoT device data. This data can be used for a wide range of purposes, from product improvement to quality management. For the further tasks, assume that you receive the data from hundreds of thousands of networkable devices worldwide.
    1. For the appliances, pseudonymous data is collected for predefined events after the user has given their consent. The appliances send various data with each status change – e.g. at the start or end of a programme or in the event of errors. You can assume that the data will be provided for you as JSON files. Keep in mind the different nature of the various types of the appliances.
        a. What challenges can you imagine when processing IoT data?
        b. Program code in preferably Python or alternatively in R or Julia, which processes the incoming device data (JSON on the following page) so that business analysts can conveniently use the data as simple, flat tables in visualisation tools such as PowerBI. In a future integration of the code, it is sufficient to load the data once a day. You are free to use any libraries/packages you like. Guide the participants through your code (approx. 15 min).
    2. Please prepare a 10-minute presentation on how your code can be integrated into a cloud environment (preferably Azure, alternatively AWS or GCP). Please answer the following questions:
        a. Where is your code stored? How do you handle new code changes?
        b. You want to work with several colleagues on further developing the code. What needs to be considered in terms of the design and management of the code?
        c. Where is your code executed? Which cloud services could be used and linked conceptually?
        d.  Are there any other cloud services that are relevant for your pipeline?
There will be a discussion with the observers after your presentation. You can act freely in terms of methodology and also prepare suitable digital documents.

                • FailureIds is an array of any length.
                • For coffee machines, each attribute has a unique ID, the value itself (Value) and the associated unit of measurement (Unit).
